STEP-BY-STEP DEPLOYMENT
═════════════════════════════════════════════════════════════════════════


STEP 1: CREATE GITHUB REPOSITORY
────────────────────────────────

1. Go to: https://github.com/new

2. Fill in the form:
   Repository name: orfeas-ai-studio
   Description: ORFEAS AI - 3D Model Generation Studio
   Visibility: PUBLIC (required for free Pages)

3. Do NOT initialize with README/gitignore (we already have commits)

4. Click: "Create repository"

5. You'll see a page with a URL like:
   https://github.com/YOUR-USERNAME/orfeas-ai-studio


STEP 2: ADD REMOTE AND PUSH CODE
─────────────────────────────────

Now push your local code to GitHub. Run these commands:

  cd c:\Users\johng\Documents\oscar

  git remote add origin https://github.com/YOUR-USERNAME/orfeas-ai-studio

  git branch -M main

  git push -u origin main

IMPORTANT: Replace "YOUR-USERNAME" with your actual GitHub username!

Example (if username is "johndoe"):
  git remote add origin https://github.com/johndoe/orfeas-ai-studio


STEP 3: ENABLE GITHUB PAGES
───────────────────────────

1. Go to your repository:
   https://github.com/YOUR-USERNAME/orfeas-ai-studio

2. Click: "Settings" (top right, near the code button)

3. Left sidebar, click: "Pages"

4. Under "Build and deployment":
   Source: Deploy from a branch
   Branch: main (select)
   Folder: / (root - should already be selected)

5. Click: "Save"

6. Wait 1-2 minutes for deployment

7. At the top, you'll see:
   "Your site is live at: https://YOUR-USERNAME.github.io/orfeas-ai-studio"

COPY THIS URL - that's your LIVE ORFEAS AI Studio!

TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════

Problem: "Repository not found" when pushing
Solution:
  • Make sure repository name is exactly: orfeas-ai-studio
  • Check your GitHub username is correct in remote URL
  • Verify you're in the right directory: c:\Users\johng\Documents\oscar

Problem: Pages not showing up in Settings
Solution:
  • Wait 5 minutes after push (GitHub needs time to process)
  • Refresh the page
  • Check the "Actions" tab to see if build succeeded

Problem: Site shows 404 error
Solution:
  • Make sure index.html or synexa-style-studio.html exists
  • You might need to rename synexa-style-studio.html to index.html
  • Or update netlify.toml to serve the correct file

Problem: API calls failing
Solution:
  • Start backend: cd backend && python -u main.py
  • Start ngrok: .\START_NGROK_TUNNEL.bat
  • Check F12 console for error messages
  • Verify ngrok URL is correct

Problem: My domain isn't working
Solution:
  • DNS changes can take 24 hours to propagate
  • Keep your github.io URL as backup
  • Verify CNAME file is in repository root
